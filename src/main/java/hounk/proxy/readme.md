代理模式的作用
    将主要业务与次要业务进行松耦合
    

代理模式本质
    本质：监控行为特征
    
案例：
    饭钱便后要洗手
    
    1.分析出主要业务与次要业务
       主要：吃饭，上厕所
       次要：洗手
            
    2.JDK代理模式实现
       1 接口角色：定义所有需要被监听的行为
       2 接口实现类：所有人
       3 通知类：次要业务进行具体实现
                 通知JVM，当前被拦截的主要业务方法与次要方法应该如何绑定执行
       4 监控对象（代理对象）
           1）被监控实例对象
           2）需要被监控行为
           3）具体通知类实例对象
